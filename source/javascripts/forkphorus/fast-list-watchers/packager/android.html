Forkphorus & Android

This is a guide to packaging Scratch projects as Android apps using Forkphorus.

WORK IN PROGRESS — Some steps are missing, and some steps are unclear. It will be improved in time.

Using the packager

Generate an HTML file using the packager.

The only important part of this step is that YOU MUST enable "Include CSP" under "Advanced Options".

Packaging it to run on Android

I'll be using Apache Cordova to do this. I have also attempted to use PhoneGap to do this, however I wasn't able to get it to work. (it seemed to inject its own scripts inside of existing scripts, which caused a lot of errors.)

  1. Download Node.js (and npm if it is not included)
  2. Get Android Studio (you might be able to get by with just ADB/SDK? Not sure)
  3. Open a Command Prompt or Terminal and execute these commands to create a Cordova App:
    1. npm install -g cordova
    2. cordova create MyApp
    3. cd MyApp
    4. cordova platform add android
  4. Delete everything inside the "www" folder created by Cordova. The template includes a lot of stuff we will not be using. (keep the "www" folder itself, but empty it out)
  5. Move the HTML file generated by the packager into the "www" folder, and rename it as "index.html"

Running it in the emulator

This step is not necessary. It is useful to test if your project actually works in Cordova.

  1. Start the emulator
  2. cordova run android
